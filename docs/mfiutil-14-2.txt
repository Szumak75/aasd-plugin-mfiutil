DDEESSCCRRIIPPTTIIOONN
     The mmffiiuuttiill utility can be used to display or modify various parameters
     on LSI MegaRAID SAS RAID controllers.  Each invocation of mmffiiuuttiill
     consists of zero or more global options followed by a command.  Commands
     may support additional optional or required arguments after the command.

     Currently three global options are supported:

     --DD _d_e_v_i_c_e
             _d_e_v_i_c_e specifies the device node of the controller to use.  If no
             device node is specified, then device will be made of the type
             and device.

     --tt _t_y_p_e
             _t_y_p_e specifies the type of the controller to work with either
             mfi(4) or mrsas(4).  If no type is specified, then the name of
             the invoked tool used to derive the type.

     --uu _u_n_i_t
             _u_n_i_t specifies the unit of the controller to work with.  If no
             unit is specified, then unit 0 is used.

     Various commands accept either or both of the two options:

     --dd      Print numeric device IDs as drive identifier.  This is the
             default.  Useful in combination with --ee to print both, numeric
             device IDs and enclosure:slot information.

     --ee      Print drive identifiers in enclosure:slot form.  See next
             paragraph on format details in context of input rather than
             output.

     Drives may be specified in two forms.  First, a drive may be identified
     by its device ID.  The device ID for configured drives can be found in
     sshhooww ccoonnffiigg.  Second, a drive may be identified by its location as
     [E_x_x:]S_y_y where _x_x is the enclosure and _y_y is the slot for each drive as
     displayed in sshhooww ddrriivveess.

     Volumes may be specified in two forms.  First, a volume may be identified
     by its target ID.  Second, on the volume may be specified by the
     corresponding _m_f_i_d_X device, such as _m_f_i_d_0.

     The mmffiiuuttiill utility supports several different groups of commands.  The
     first group of commands provide information about the controller, the
     volumes it manages, and the drives it controls.  The second group of
     commands are used to manage the physical drives attached to the
     controller.  The third group of commands are used to manage the logical
     volumes managed by the controller.  The fourth group of commands are used
     to manage the drive configuration for the controller.  The fifth group of
     commands are used to manage controller-wide operations.

     The informational commands include:

     vveerrssiioonn
             Displays the version of mmffiiuuttiill.

     sshhooww aaddaapptteerr
             Displays information about the RAID controller such as the model
             number.

     sshhooww bbaatttteerryy
             Displays information about the battery from the battery backup
             unit.

     sshhooww ccoonnffiigg
             Displays the volume and drive configuration for the controller.
             Each array is listed along with the physical drives the array is
             built from.  Each volume is listed along with the arrays that the
             volume spans.  If any hot spare drives are configured, then they
             are listed as well.

     sshhooww ddrriivveess
             Lists all of the physical drives attached to the controller.

     sshhooww eevveennttss [--cc _c_l_a_s_s] [--ll _l_o_c_a_l_e] [--nn _c_o_u_n_t] [--vv] [_s_t_a_r_t [_s_t_o_p]]
             Display entries from the controller's event log.  The controller
             maintains a circular buffer of events.  Each event is tagged with
             a class and locale.

             The _c_l_a_s_s parameter limits the output to entries at the specified
             class or higher.  The default class is “warn”.  The available
             classes from lowest priority to highest are:

             ddeebbuugg   Debug messages.

             pprrooggrreessss
                     Periodic progress updates for long-running operations
                     such as background initializations, array rebuilds, or
                     patrol reads.

             iinnffoo    Informational messages such as drive insertions and
                     volume creations.

             wwaarrnn    Indicates that some component may be close to failing.

             ccrriitt    A component has failed, but no data is lost.  For
                     example, a volume becoming degraded due to a drive
                     failure.

             ffaattaall   A component has failed resulting in data loss.

             ddeeaadd    The controller itself has died.

             The _l_o_c_a_l_e parameter limits the output to entries for the
             specified part of the controller.  The default locale is “all”.
             The available locales are “volume”, “drive”, “enclosure”,
             “battery”, “sas”, “controller”, “config”, “cluster”, and “all”.

             The _c_o_u_n_t parameter is a debugging aid that specifies the number
             of events to fetch from the controller for each low-level
             request.  The default is 15 events.

             By default, matching event log entries from the previous shutdown
             up to the present are displayed.  This range can be adjusted via
             the _s_t_a_r_t and _s_t_o_p parameters.  Each of these parameters can
             either be specified as a log entry number or as one of the
             following aliases:

             nneewweesstt  The newest entry in the event log.

             oollddeesstt  The oldest entry in the event log.

             cclleeaarr   The first entry since the event log was cleared.

             sshhuuttddoowwnn
                     The entry in the event log corresponding to the last time
                     the controller was cleanly shut down.

             bboooott    The entry in the event log corresponding to the most
                     recent boot.

     sshhooww ffiirrmmwwaarree
             Lists all of the firmware images present on the controller.

     sshhooww ffoorreeiiggnn
             Displays detected foreign configurations on disks for importation
             or removal.

     sshhooww llooggssttaattee
             Display the various sequence numbers associated with the event
             log.

     sshhooww ppaattrrooll
             Display the status of the controller's patrol read operation.

     sshhooww pprrooggrreessss
             Report the current progress and estimated completion time for
             active operations on all volumes and drives.

     sshhooww vvoolluummeess
             Lists all of the logical volumes managed by the controller.

     The physical drive management commands include:

     ffaaiill _d_r_i_v_e
             Mark _d_r_i_v_e as failed.  _D_r_i_v_e must be an online drive that is part
             of an array.

     ggoooodd _d_r_i_v_e
             Mark _d_r_i_v_e as an unconfigured good drive.  _D_r_i_v_e must not be part
             of an existing array.

     rreebbuuiilldd _d_r_i_v_e
             Mark a failed _d_r_i_v_e that is still part of an array as a good
             drive suitable for a rebuild.  The firmware should kick off an
             array rebuild on its own if a failed drive is marked as a rebuild
             drive.

     ssyyssppdd _d_r_i_v_e
             Present the drive to the host operating system as a disk SYSPD
             block device in the format /dev/mfisyspdX.  Clear this flag with
             ggoooodd _d_r_i_v_e

     ddrriivvee pprrooggrreessss _d_r_i_v_e
             Report the current progress and estimated completion time of
             drive operations such as rebuilds or patrol reads.

     ddrriivvee cclleeaarr _d_r_i_v_e {start | stop}
             Start or stop the writing of all 0x00 characters to a drive.

     ssttaarrtt rreebbuuiilldd _d_r_i_v_e
             Manually start a rebuild on _d_r_i_v_e.

     aabboorrtt rreebbuuiilldd _d_r_i_v_e
             Abort an in-progress rebuild operation on _d_r_i_v_e.  It can be
             resumed with the ssttaarrtt rreebbuuiilldd command.

     llooccaattee _d_r_i_v_e {on | off}
             Change the state of the external LED associated with _d_r_i_v_e.

     The logical volume management commands include:

     ccaacchhee _v_o_l_u_m_e [_s_e_t_t_i_n_g [_v_a_l_u_e] [...]]
             If no _s_e_t_t_i_n_g arguments are supplied, then the current cache
